# Docs — Context Document Generation

Generate structured context documents that help Claude understand this project deeply.

## What Gets Generated

### Automated (codebase scan — no human input needed)

1. **platform-docs.md** — Every feature, screen, route, API endpoint
   - Scan all page/route files, API handlers, components
   - Document what each screen does, its URL, key interactions

2. **architecture.md** — Data models, APIs, integrations, tech stack
   - Read schema files, config, package.json/pyproject.toml
   - Document database models, API structure, third-party integrations

3. **styleguide.md** — Colors, fonts, spacing, component patterns
   - Read tailwind config, CSS files, theme definitions
   - Document the design system, spacing scale, color palette

### Interview-Based (asks the founder 3-5 questions)

4. **ICPs.md** — Ideal Customer Profiles
   - Ask: Who are your first 10 users? What do they use now? What job are they hiring this for?
   - Document user segments, their context, their alternatives

5. **roadmap.md** — Vision, priorities, what NOT to build
   - Ask: What's the vision? What are you NOT building? What would make you kill this project?
   - Document priorities, anti-goals, kill criteria

## How To Generate

For each automated doc, read the relevant code files and synthesize. Write comprehensive but concise documents.

For interview docs, ask the user focused questions (3-5 max per doc). Don't over-interview.

## Output

Write all documents to the `documents/` directory in the project root.

Add a metadata comment at the top of each file:
```
<!-- Generated by /docs | [ISO date] | [git SHA] -->
```

## Refresh Mode

If `documents/` already exists, check what changed since last generation:
- Run `git diff --name-only` against the SHA in the metadata comment
- Only regenerate docs whose trigger files changed (see rhino.yml `docs.trigger_*` patterns)
- For interview docs: skip refresh (those don't change with code)

Tell the user: "Run `/docs --refresh` to update only changed docs."
